# Crypto AI Trading Bot

بوت تداول عملات رقمية تجريبي يعتمد على تحليل فني قابل للتفسير وإدارة مخاطر صارمة. المشروع يبدأ في وضع **paper trading** افتراضياً حتى تختبر الاستراتيجية جيداً قبل التفكير في أي تنفيذ حقيقي.

> تنبيه مهم: لا يوجد بوت يضمن صفقات رابحة دائماً. الهدف هنا هو تحسين جودة القرار وتقليل المخاطر باستخدام قواعد واضحة مثل وقف الخسارة، جني الأرباح، وحد أقصى للخسارة اليومية.

## المميزات

- تحليل السوق باستخدام متوسطات متحركة، RSI، وفلتر تقلبات.
- محرك قرارات هجين لا يحتاج مكتبات ثقيلة، ويمكن استبداله لاحقاً بنموذج تعلم آلي مدرّب.
- إدارة رأس مال تحدد حجم الصفقة بناءً على نسبة مخاطرة ثابتة.
- أوامر وقف خسارة وجني أرباح لكل صفقة.
- حد أقصى للخسارة اليومية يمنع الاستمرار عند سوء ظروف السوق.
- تكامل اختياري مع `ccxt` لجلب بيانات السوق، مع إبقاء التنفيذ الورقي كخيار آمن.

## التشغيل السريع

### 1) جهّز البيئة

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
```

على Windows استخدم:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
```

### 2) شغّل الاختبارات

```bash
pytest
```

### 3) شغّل البوت محلياً بدون إنترنت

هذا الوضع يستخدم بيانات Demo مولّدة، ولا يحتاج مفاتيح API أو اتصال بمنصة تداول:

```bash
crypto-ai-bot --demo
```

أو مباشرة من السورس بدون تثبيت السكربت:

```bash
PYTHONPATH=src python -m crypto_ai_bot.cli --demo
```

ستظهر نتيجة JSON فيها الإشارة وخطة الأمر الورقي، مثل `buy` أو `hold` مع وقف الخسارة وجني الأرباح إن وُجدت صفقة.

### 4) جلب بيانات مباشرة عبر ccxt

الأوامر تظل Paper Trading، لكن الشموع تأتي من المنصة المختارة:

```bash
pip install -e '.[live]'
SYMBOL=BTC/USDT TIMEFRAME=1h EXCHANGE_ID=binance crypto-ai-bot --live-data
```


## تشغيل مباشر على Oracle Cloud بدون virtualenv

لو عاوز تشغله مباشرة من نسخة الكود، من غير إنشاء بيئة Python ومن غير تثبيت الحزمة، استخدم السكربت الجاهز:

```bash
git clone <repo-url> crypto-ai-bot
cd crypto-ai-bot
python3 run_bot.py --demo
```

وضع `--demo` لا يحتاج إنترنت ولا مفاتيح API. لو هتستخدم بيانات مباشرة من منصة تداول، ثبّت `ccxt` على النظام ثم شغّل:

```bash
python3 -m pip install --user ccxt
SYMBOL=BTC/USDT TIMEFRAME=1h EXCHANGE_ID=binance python3 run_bot.py --live-data
```

لتشغيله كخدمة مجدولة على Oracle Cloud، انسخ ملف `oracle-cloud-systemd.service.example` وعدّل `WorkingDirectory` و `ExecStart` حسب مكان المشروع على السيرفر.

## إعدادات مهمة

يمكن ضبط الإعدادات عبر متغيرات البيئة:

| المتغير | الافتراضي | الوصف |
| --- | --- | --- |
| `SYMBOL` | `BTC/USDT` | زوج التداول |
| `TIMEFRAME` | `1h` | الإطار الزمني للشموع |
| `STARTING_BALANCE` | `10000` | رأس المال الورقي |
| `RISK_PER_TRADE` | `0.01` | نسبة المخاطرة في الصفقة الواحدة |
| `MAX_DAILY_LOSS` | `0.03` | حد الخسارة اليومية |
| `STOP_LOSS_PCT` | `0.02` | وقف الخسارة |
| `TAKE_PROFIT_PCT` | `0.04` | جني الأرباح |
| `MIN_CONFIDENCE` | `0.58` | أقل ثقة لقبول إشارة تداول |

## قبل التداول الحقيقي

1. اختبر الاستراتيجية على بيانات تاريخية لفترة طويلة.
2. شغلها Paper Trading على عدة أسابيع على الأقل.
3. أضف مفاتيح API بصلاحيات محدودة وبدون سحب أموال.
4. ابدأ بمبلغ صغير جداً، وراقب الانزلاق السعري والرسوم.
5. لا تخاطر بأموال لا تتحمل خسارتها.
